变量¶

Julia的变量是绑定的一个值

  • 变量名称不仅仅可以是ASCII字符，也可以是其他的Unicode字符，甚至是中文
  • 像其他动态语言一样，可以无需声明直接赋值
  • 可以输入\+符号名称的方式输入更多的Unicode数学字符，如\alpha后按tab键就会出现$\alpha$ ### 变量的命名规范
  • 变量名尽量小写
  • 类型和没款名首字母大写，单词间使用驼峰式分割
  • 在几个单词不易区分时才以_分割，一般不鼓励使用下划线_
  • 函数名和宏名使用小写字母，不使用下户线
  • 修改参数的函数结尾使用!，这样的函数被称为mutating functions或in-place functions

小练习3：随机行走 随机行走（Random Walk）是在物理，金融，化学，生物等领域都会遇到的基本问题，我们这里将模拟最简单的一种：随机在一维坐标下行走。通过这个例子来了解基本的Julia程序的编写过程。

In [9]:
"""
    walk(t) -> trajectory
模拟随机行走
"""
function walk(t)
    x = 0 #初始位置
    trajectory = [x] # 初始化轨迹

    for t in 1:t 
        if rand() < 0.5
            x += 1
        else
            x -= 1
        end
        push!(trajectory,x)
    end
    return trajectory   
end

面向对象¶

一切都是对象¶

Julia里所有的一切都是对象，Julia语言也是一个支持面向对象的语言。但是Julia的对象皆为类型的实例，而非class的实例，Julia没有class，但是有更轻量级的类型。

定义一个复数类型¶

复数是很多数值计算和数据分析算法都会用到的类型，然而一般的处理器上并不是自带复数类型的，我们往往使用两倍精度的浮点数来模拟一个复数，例如在C语言里，双精度的复数可以写作：

double _Complex a;

或者在C++中，双精度的复数可以写作：

std::complex<double> a;

在Python里，没有显示类型，但是我们可以使用j:

1 + 1.j

而实际上和C/C++一样，Julia的复数类型也是纯Juia实现的，我们这里会简单的实现一个复数类型MyComplex。Julia的类型使用struct关键字，然后用end表示这一段表达式的结束。每个Julia类型有一个默认的构造函数，这个构造函数的变量即为类型声明的成员。

In [68]:
struct MyComplex
    real::Float64
    imag::Float64
end

# 一个复数就是 MyComplex 类型的一个实例，也是一种对象
a = MyComplex(1.0,2.0)

MyComplex(1.0, 2.0)
In [69]:

但是仅仅声明了类型还远远不够，我们还需要对复数定义复数运算，方便起见我们这里仅重载*算符： $$(a+b\cdot i)(c+d\cdot i)=(acc-bd)+(add+bc)i$$ 首先我们需要将要重载的东西从Base模块中拿出来（而不是自己声明一个新的，为什么？），在Julia里，运算符只是一种特殊的函数，并不会被特别对待

In [74]:
import Base: *
function *(a::MyComplex,b::MyComplex)
    MyComplex(a.real * b.real - a.imag * b.imag, a.real * b.imag + a.imag * b.real)
end

任意精度的复数类型¶

我们已经有了一个简单的复数类型，但是实际上真正在用的时候，我们可能会需要不同精度的复数类型，例如使用32位浮点精度的复数类型。为了能够使用不同的复数类型，我们需要使用参数类型，而复数的 实部和虚部都是实数，我们还需要限制其类型范围

In [80]:
struct MyComplex2{T <: Real} # <: 表示T一定是一个Real类型

    imag::T 
end
In [81]:

实际上可以定义一些自己构造的函数，在Julia里因为是没有class的，除了构造函数以外的方法都不能卸载类型声明内部。而一旦在类型声明中声明了一个自己的构造函数，默认的构造将被覆盖。如下例：

    new{T}(real,0)
end

多重派发和Julia的面向对象¶

Julia语言是没有class的，但这并不意味着Julia无法面对对象，Julia对象的方法（method）通过多重派发和类型树进行分配，而不依赖于class这种结构。

定义自己的字符串字面量¶

字符串的字面量是区分不同类型的字符串的一种非常方便的方法，在Python里，有正则表达式字面量r"(.*)"，格式化字面量f"hello {who}"。而在Julia里，则可以通过宏定义自己的字符串字面量，只需声明以_str为结尾的宏即可

In [92]:
struct MyStr <: AbstractString
    data::String
end

Base.show(io::IO,s::MyStr) = print(io, "My String is:", s.data)

macro my_str(s)    #定义宏
    MyStr(s)
end

my"hello!" #typeof(my"hello!")

My String is:hello!

这大大的方便了需要做大量字符串处理的任务，例如生物信息等。此外这也使得Julia很容易在文档里支持LaTex，Markdown等多种不同的格式，并且按照统一的接口（Interface）给它们派发不同的方法。
